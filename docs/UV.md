# 🛠️ The Ultimate `uv` Guide

**uv** is an extremely fast Python package installer and resolver, written in Rust. It is designed as a drop-in replacement for `pip`, `pip-tools`, and `virtualenv`.

## 1. Local Setup

If you haven't already, install `uv` on your host machine to manage the project without always needing Docker.

- **macOS/Linux:** `curl -LsSf https://astral.sh/uv/install.sh | sh`
- **Windows:** `powershell -c "irm https://astral.sh/uv/install.ps1 | iex"`

## 2. Core Workflows

### Creating a New Project

If starting from scratch:

```bash
uv init
```

### Managing Dependencies

This is the most common task. **uv** handles the `pyproject.toml` and `uv.lock` automatically.

| Action               | Command               | What it does                                         |
| -------------------- | --------------------- | ---------------------------------------------------- |
| **Add a package**    | `uv add pandas`       | Adds to `pyproject.toml` and updates `uv.lock`.      |
| **Add dev-only**     | `uv add --dev pytest` | Adds to a separate development-only group.           |
| **Remove package**   | `uv remove pandas`    | Cleans up the manifest and lockfile.                 |
| **Sync environment** | `uv sync`             | Makes your local `.venv` match the lockfile exactly. |

💡 **Tips:** `--no-sync` avoid installing libraries locally in .venv. It will only update pyproject.toml and uv.lock. Usefull to avoid duplication of installation and save disk space. Do not use this parameter if you are coding in local without launching containers.

## 3. The Magic of `uv.lock`

The `uv.lock` file is a **cross-platform** lockfile. Unlike a `requirements.txt` generated by `pip freeze`, `uv.lock`:

- Is machine-readable and deterministic.
- Ensures that your teammate on a Mac and your Docker container on Linux get the exact same dependency tree.
- **Never edit this file manually.**

## 4. Python Version Management

You can edit pyproject.toml and update python version on the fly.
```toml
[project]
name = "science-repo"
...
requires-python = "==3.13.*"
...
```
Then update uv.lock with
```bash
uv lock
```

## 5. Using `uv` with Docker

Current Dockerfile uses `uv sync`. Here is how that interaction works:

1. **Cache Efficiency:** By copying `uv.lock` and `pyproject.toml` first, Docker caches your environment and install binaries adapted for it.
2. **The Interface:** When you want to update your Docker environment:

- Run `uv add <package> --no-sync` locally.
- Run `docker compose build` or `docker compose up --build`
- Docker sees the updated `uv.lock` and triggers a re-install.

## 6. Pro Tips for Scientists 🔬

### Add git as dependency
  To add a git repository as a dependencies execute this command

```bash
uv add "<package-name> @ <git-url>"

# Exemple
uv add "streamlit-drawable-canvas @ git+https://github.com/Sourav-Ganguli/streamlit-drawable-canvas.git"
```

### Run scripts instantly

If you have a single-file script that needs a specific library, you don't even need a project:

```bash
uv run --with numpy script.py
```

### Clean your cache

If you're running low on disk space from many different experiments:

```bash
uv cache clean
```

### View dependency tree

To see why a specific sub-dependency was installed:

```bash
uv tree
```
